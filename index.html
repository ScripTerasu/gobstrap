Vista General

Una revisión del uso, la infraestructura y el enfoque en un desarrollo web mejor, más rápido y más sólido de Gobstrap.

Cómo usarlo

Para comenzar a usar Gobstrap, basta con llamar al archivo compilado dentro del <header>.

<link href="../dist/css/gobstrap.min.css" rel="stylesheet">

Doctype HTML5

Gobstrap usa ciertos elementos HTML y propiedades CSS que requieren el uso del doctype HTML5. Debes incluirlo en el inicio de todos tus proyectos.

<!DOCTYPE html> 
<html lang="en"> 
... 
</html>

Mobile first

Gobstrap es mobile friendly desde el principio. En vez de añadir estilos especiales para dispositivos móviles, estos están incluídos en su núcleo. Gobstrap es mobile first, lo que significa que se pueden encontrar estilos para dispositivos móviles en toda la librería, en vez de separados en un archivo especial.

Para asegurar la correcta visualización e interacciones, añade el meta tag de viewport a tu <head>.

<meta name="viewport" content="width=device-width, initial-scale=1">

Puedes desabilitar el zoom en dispositivos móviles añadiendo user-scalable=no al meta tag viewport. Esto hará que los usuarios sólo puedan hacer scroll, resultando en un sitio que se parece más a una aplicación nativa. De todas maneras, no recomendamos esto en todos los sitios, así que procura usarlo con prudencia.

<meta name="viewport" content="width=device-width, initial-scale=1, maximum-scale=1, user-scalable=no">

Tipografías y links

Gobstrap establece estilos básicos globales, de tipografías y links. Principalmente:

  • Fija background-color: #fff; en el body
  • Usa los atributos @font-family-base, @font-size-base, y @line-height-base como nuestra base tipográfica
  • Fija el color global de los links via @link-color y aplica subrayado sólo en el estado :hover

Estos estilos pueden encontrarse en el archivo scaffolding.less.

Normalize.css

Para una visualización lo más consistente posible en distintos navegadores (cross-browsing), Gobstrap usa Normalize.css, un proyecto de Nicolas Gallagher y Jonathan Neal.

Contenedores

Gobstrap requiere un elemento contenedor para envolver los contenidos del sitio y alojar el sistema de grillas. Puedes escoger uno de dos contenedores para usar en tus proyectos. Ten en cuenta que debido al padding y otros elementos, ninguno de los contenedores son anidables.

Usa .container para un contenedor responsivo de ancho fijo.

<div class="container">
  ...
</div>

Usa .container-fluid para un contenedor del ancho completo del viewport.

<div class="container-fluid">
  ...
</div>

Sistema de Grilla

Gobstrap inluye una grilla fluida, responsiva y mobile first que se escala apropiadamente hasta 12 columnas a medida que el dispositivo o la ventana del navegador crece. Esta grilla incluye clases predefinidas para facilitar las opciones de layout, y también mixins para generar layouts más semánticos.

Introducción

Las grillas son usadas para la creación de layouts en sitios web mediante una serie de filas y columnas que alojan el contenido. El sistema de Gobstrap funciona de la siguiente manera:

  • Las filas deben ser incluidas dentro de un .container (ancho fijo) o .container-fluid (ancho completo) para alineamiento y padding correcto.
  • Usa las filas para crear grupos horizontales de columnas.
  • El contenido siempre debe ser incluido dentro de columnas, y sólo columnas pueden ser elementos inmediatamente descendientes de las filas.
  • Hay clases predefinidas de la grilla, como .row y .col-xs-4 para una rápida creación de layouts. También pueden usarse mixins de less para layouts más semánticos.
  • Las columnas crean "gutters" (espacios entre las columnas) mediante padding. Ese padding es compensado en la primera y la última columna de cada fila, mediante un margen negativo en las .row.
  • Ese margen negativo es la razón de por qué los ejemplos más abajo están fuera de la fila. Es para que el contenido en las columnas de la grilla esté alineado con el contenido fuera de la grilla.
  • Las columnas de la grilla se crean especificando el número de columnas que quieres ocupar, de un total del 12. Por ejemplo, 3 columnas iguales usarían tres .col-xs-4.
  • Si más de 12 columnas son puestas en una sola fila, cada grupo de columnas extra se acomplarán en una nueva línea, como un grupo.
  • Las clases de la grilla se aplican a dispositivos con anchos de pantalla iguales o más grandes que los breakpoints (puntos de quiebre), y sobreescriben clases para pantallas más pequeñas. Entonces, aplicar cualquier clase .col-md- a un elemento, no sólo afectará su estilo en dispositivos medianos, si no también en dispositivos grandes, si no tiene aplicado algún .col-lg-.

Mira los ejemplos para aplicar correctamente estos principios a tu código.

Media queries

Gobstrap usa los siguientes media queries en sus archivos Less para crear los puntos de quiebre del sistema de grilla.

/* Dispositivos muy pequeños (teléfonos, menores a 740px) */
/* No hay media queries, ya que este es el predeterminado en Gobstrap */

/* Dispositivos pequeños (tablets) (740px y más) */
@media (min-width: @screen-sm-min) { ... }

/* Dispositivos medianos (computadores de escritorio/laptops, 1020px y más) */
@media (min-width: @screen-md-min) { ... }

/* Dispositivos grandes (pantallas grandes, 1300px y más) */
@media (min-width: @screen-lg-min) { ... }

Ocasionalmente extendemos estas media queries e incluimos max-width para limitar el CSS a un conjunto más reducido de dispositivos.

@media (max-width: @screen-xs-max) { ... }
@media (min-width: @screen-sm-min) and (max-width: @screen-sm-max) { ... }
@media (min-width: @screen-md-min) and (max-width: @screen-md-max) { ... }
@media (min-width: @screen-lg-min) { ... }

Contenedor tarjeta

Para cualquier elemento sin estilo y sin un contenedor, se recomienda usarlo dentro de un div con la clase .tarjeta, que le dará un padding, color de fondo y borde acorde a los estilos de gobstrap. De esta manera se puede presentar información fácilmente, contener otros elementos o construir nuevos módulos a partir de este elemento base.

Lorem ipsum dolor sit amet
<div class="bs-example" style="background:#f1f1f1">
  <div class="tarjeta">Lorem ipsum dolor sit amet</div>
</div>

Atención con el uso

Debido a sus características de alto y ancho, los módulos .tarjeta-full y .tarjeta-horizontal están diseñadas para ser usadas en condiciones específicas: .tarjeta-full dentro de un .col-md-12 y .tarjeta-horizontal dentro de .col-md-12 o .col-md-8. Para usarlo en otras opciones de grilla, es necesario revisar y adaptar -cuando corresponda- los estilos en gobstrap.less.
