Vista general

Una revisión del uso, la infraestructura y el enfoque en un desarrollo web mejor, más rápido y más sólido de Gobstrap.

Cómo usarlo

Para comenzar a usar Gobstrap, basta con llamar al archivo compilado dentro del <header>.

<link href="../dist/css/gobstrap.min.css" rel="stylesheet">

Doctype HTML5

Gobstrap usa ciertos elementos HTML y propiedades CSS que requieren el uso del doctype HTML5. Debes incluirlo en el inicio de todos tus proyectos.

<!DOCTYPE html>
<html lang="en">
  ...
</html>

Mobile first

Gobstrap es mobile friendly desde el principio. En vez de añadir estilos especiales para dispositivos móviles, estos están incluídos en su núcleo. Gobstrap es mobile first, lo que significa que se pueden encontrar estilos para dispositivos móviles en toda la librería, en vez de separados en un archivo especial.

Para asegurar la correcta visualización e interacciones, añade el meta tag de viewport a tu <head>.

<meta name="viewport" content="width=device-width, initial-scale=1">

Puedes desabilitar el zoom en dispositivos móviles añadiendo user-scalable=no al meta tag viewport. Esto hará que los usuarios sólo puedan hacer scroll, resultando en un sitio que se parece más a una aplicación nativa. De todas maneras, no recomendamos esto en todos los sitios, así que procura usarlo con prudencia.

<meta name="viewport" content="width=device-width, initial-scale=1, maximum-scale=1, user-scalable=no">

Contenedores

Gobstrap requiere un elemento contenedor para envolver los contenidos del sitio y alojar el sistema de grillas. Puedes escoger uno de dos contenedores para usar en tus proyectos. Ten en cuenta que debido al padding y otros elementos, ninguno de los contenedores son anidables.

Sistema de Grilla

Gobstrap inluye una grilla fluida, responsiva y mobile first que se escala apropiadamente hasta 12 columnas a medida que el dispositivo o la ventana del navegador crece. Esta grilla incluye clases predefinidas para facilitar las opciones de layout, y también mixins para generar layouts más semánticos.

Introducción

Las grillas son usadas para la creación de layouts en sitios web mediante una serie de filas y columnas que alojan el contenido. El sistema de Gobstrap funciona de la siguiente manera:

  • Las filas deben ser incluidas dentro de un .container (ancho fijo) o .container-fluid (ancho completo) para alineamiento y padding correcto.
  • Usa las filas para crear grupos horizontales de columnas.
  • El contenido siempre debe ser incluido dentro de columnas, y sólo columnas pueden ser elementos inmediatamente descendientes de las filas.
  • Hay clases predefinidas de la grilla, como .row y .col-xs-4 para una rápida creación de layouts. También pueden usarse mixins de less para layouts más semánticos.
  • Las columnas crean "gutters" (espacios entre las columnas) mediante padding. Ese padding es compensado en la primera y la última columna de cada fila, mediante un margen negativo en las .row.
  • Ese margen negativo es la razón de por qué los ejemplos más abajo están fuera de la fila. Es para que el contenido en las columnas de la grilla esté alineado con el contenido fuera de la grilla.
  • Las columnas de la grilla se crean especificando el número de columnas que quieres ocupar, de un total del 12. Por ejemplo, 3 columnas iguales usarían tres .col-xs-4.
  • Si más de 12 columnas son puestas en una sola fila, cada grupo de columnas extra se acomplarán en una nueva línea, como un grupo.
  • Las clases de la grilla se aplican a dispositivos con anchos de pantalla iguales o más grandes que los breakpoints (puntos de quiebre), y sobreescriben clases para pantallas más pequeñas. Entonces, aplicar cualquier clase .col-md- a un elemento, no sólo afectará su estilo en dispositivos medianos, si no también en dispositivos grandes, si no tiene aplicado algún .col-lg-.

Mira los ejemplos para aplicar correctamente estos principios a tu código.

Media queries

Gobstrap usa los siguientes media queries en sus archivos Less para crear los puntos de quiebre del sistema de grilla.

/* Dispositivos muy pequeños (teléfonos, menores a 740px) */
/* No hay media queries, ya que este es el predeterminado en Gobstrap */

/* Dispositivos pequeños (tablets) (740px y más) */
@media (min-width: @screen-sm-min) { ... }

/* Dispositivos medianos (computadores de escritorio/laptops, 1020px y más) */
@media (min-width: @screen-md-min) { ... }

/* Dispositivos grandes (pantallas grandes, 1300px y más) */
@media (min-width: @screen-lg-min) { ... }

Ocasionalmente extendemos estas media queries e incluimos max-width para limitar el CSS a un conjunto más reducido de dispositivos.

@media (max-width: @screen-xs-max) { ... }
@media (min-width: @screen-sm-min) and (max-width: @screen-sm-max) { ... }
@media (min-width: @screen-md-min) and (max-width: @screen-md-max) { ... }
@media (min-width: @screen-lg-min) { ... }

Mixins y variables de Less

Además de las clases incluidas de grilla para construir rápidamente un layout, Gobstrap incluye mixins y variables en Less para crear layouts propios, simples y semánticos.

Variables

Estas variables determinan el número de columnas, el ancho del gutter (espacio entre columnas) y el punto de media query en el que las columnas comienzan a flotar. Gobstrap usa estas variables para generar las clases predefinidas documentadas anteriormente, y también los mixins personalizados listados más abajo.

@grid-columns:              12;
@grid-gutter-width:         30px;
@grid-float-breakpoint:     768px;

Mixins

Los mixins son usados en conjunto con las variables de la grilla para generar CSS semántico para columnas individuales.

// Crear un wrapper para un conjunto de columnas
.make-row(@gutter: @grid-gutter-width) {
  // Limpiar el flujo de las columnas flotantes
  .clearfix();

  @media (min-width: @screen-sm-min) {
    margin-left:  (@gutter / -2);
    margin-right: (@gutter / -2);
  }

  // Margen negativo en las rows anidadas para alinear el contenido de las columnas
  .row {
    margin-left:  (@gutter / -2);
    margin-right: (@gutter / -2);
  }
}

// Generar las columnas extra pequeñas
.make-xs-column(@columns; @gutter: @grid-gutter-width) {
  position: relative;
  // Prevenir que las columnas se colapsen cuando están vacías
  min-height: 1px;
  // Gutter interior mediante padding
  padding-left:  (@gutter / 2);
  padding-right: (@gutter / 2);

  // Calcular el ancho basado en el número de columnas disponibles
  @media (min-width: @grid-float-breakpoint) {
    float: left;
    width: percentage((@columns / @grid-columns));
  }
}

La escala tipográfica está basada en dos variables de Less declaradas en variables.less: @font-size-base y @line-height-base. Usando Less

Gobstrap está construido en Less, un preprocesador de CSS con otras funcionalidades extras como variables, mixins y funciones para compilar CSS. Aquellos que prefieran usar los archivos fuente Less en vez de los CSS compilados pueden usar libremente una basta cantidad de variables y mixines que son usados a través de todo el framework.

Las variables y mixins de grilla están cubiertas en la sección del Sistema de grilla.

Compilando Gobstrap

Gobstrap puede ser usado de al menos dos maneras: con el CSS compilado o con los archivos fuente Less. Para compilar los archivos Less, consulta la sección Getting Started (?) para saber cómo instalar el entorno de desarrollo para correr los comandos necesarios.

Herramientas de compilación de terceros pueden funcionar con Gobstrap, pero no están soportadas por equipo central.

Variables

Las variables son usadas a lo largo del projecto completo de manera de centralizar y compartir valores usados comunmente como colores, espaciados y propiedades de fuentes y módulos. Para un desglose completo, lee el Customizer.

Cómo usarlos

Para una correcta visualización, todos los íconos requieren una clase base y una clase específica del ícono. Para usarlos, copia el código correspondiente. Asegúrate de dejar un espacio entre el ícono y el texto para un padding correcto.

No los mezcles con otros componentes

Las clases de íconos no pueden ser directamente combinadas con otros componentes. No deben ser usados con otras clases en el mismo elemento. En vez de eso, agrega un <span> anidado y aplica el las clases de ícono al <span>.

Sólo para uso en elementos vacíos

Las clases de íconos deben ser usadas sólo en elementos que no contienen texto y no tienen elementos dentro.

Cambiar la ubicación de las tipografías

Gobstrap asume que los archivos de la tipgorafía están en la carpeta ../fonts/, relativa a los archivos CSS compilados. Mover o renombrar esos archivos de la tipografía implica Bootstrap assumes icon font files will be located in the ../fonts/ directory, relative to the compiled CSS files. Moving or renaming those font files means updating the CSS in one of three ways:

  • Change the @icon-font-path and/or @icon-font-name variables in the source Less files.
  • Utilize the relative URLs option provided by the Less compiler.
  • Change the url() paths in the compiled CSS.

Use whatever option best suits your specific development setup.

Alignment

By default, a dropdown menu is automatically positioned 100% from the top and along the left side of its parent. Add .dropdown-menu-right to a .dropdown-menu to right align the dropdown menu.

May require additional positioning

Dropdowns are automatically positioned via CSS within the normal flow of the document. This means dropdowns may be cropped by parents with certain overflow properties or appear out of bounds of the viewport. Address these issues on your own as they arise.

Deprecated .pull-right alignment

As of v3.1.0, we've deprecated .pull-right on dropdown menus. To right-align a menu, use .dropdown-menu-right. Right-aligned nav components in the navbar use a mixin version of this class to automatically align the menu. To override it, use .dropdown-menu-left.

Handling borders

Due to the specific HTML and CSS used to justify buttons (namely display: table-cell), the borders between them are doubled. In regular button groups, margin-left: -1px is used to stack the borders instead of removing them. However, margin doesn't work with display: table-cell. As a result, depending on your customizations to Bootstrap, you may wish to remove or re-color the borders.

IE8 and borders

Internet Explorer 8 doesn't render borders on buttons in a justified button group, whether it's on <a> or <button> elements. To get around that, wrap each button in another .btn-group.

Navbar

Default navbar

Navbars are responsive meta components that serve as navigation headers for your application or site. They begin collapsed (and are toggleable) in mobile views and become horizontal as the available viewport width increases.

Overflowing content

Since Bootstrap doesn't know how much space the content in your navbar needs, you might run into issues with content wrapping into a second row. To resolve this, you can:

  1. Reduce the amount or width of navbar items.
  2. Hide certain navbar items at certain screen sizes using responsive utility classes.
  3. Change the point at which your navbar switches between collapsed and horizontal mode. Customize the @grid-float-breakpoint variable or add your own media query.

Requires JavaScript

If JavaScript is disabled and the viewport is narrow enough that the navbar collapses, it will be impossible to expand the navbar and view the content within the .navbar-collapse.

Changing the collapsed mobile navbar breakpoint

The navbar collapses into its vertical mobile view when the viewport is narrower than @grid-float-breakpoint, and expands into its horizontal non-mobile view when the viewport is at least @grid-float-breakpoint in width. Adjust this variable in the Less source to control when the navbar collapses/expands. The default value is 768px (the smallest "small" or "tablet" screen).

Atención con el uso

Debido a sus características de alto y ancho, los módulos .tarjeta-full y .tarjeta-horizontal están diseñadas para ser usadas en condiciones específicas: .tarjeta-full dentro de un .col-md-12 y .tarjeta-horizontal dentro de .col-md-12 o .col-md-8. Para usarlo en otras opciones de grilla, es necesario revisar y adaptar -cuando corresponda- los estilos en gobstrap.less.
